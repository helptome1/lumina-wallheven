# Wallhaven Tauri 功能文档

本文档记录当前 `wallhaven-tauri` 版本已经具备的产品能力、页面结构、数据流与关键交互。它面向后续维护者和设计/开发协作场景，重点解释“现在能做什么”和“功能是如何组织的”。

## 1. 产品定位

`wallhaven-tauri` 是一个基于 Wallhaven API 的桌面壁纸浏览与下载客户端。前端使用 Vue 3、Pinia、Vue Router 和 Tailwind CSS，桌面能力由 Tauri/Rust 提供。

核心目标：

- 快速浏览不同类型的 Wallhaven 壁纸。
- 支持搜索、筛选、无限滚动加载。
- 支持壁纸详情查看、放大缩放、拖动画布。
- 支持收藏、下载、本地文件定位和删除。
- 通过 Tauri 后端代理 API 与图片请求，规避浏览器 CORS 和 CDN 防盗链限制。

## 2. 信息架构

应用主框架由 `AppShell` 承载，左侧为固定导航栏，右侧为内容区域。

当前路由：

| 路径 | 页面 | 用途 | 关键类型参数 |
| --- | --- | --- | --- |
| `/hot` | 热门推荐 | 默认首页，展示综合热门壁纸 | `categories=111` |
| `/acg` | 动漫精选 | 展示动漫/插画方向壁纸 | `categories=010` |
| `/anime` | 重定向 | 兼容旧入口 | 重定向到 `/acg` |
| `/people` | 人物精选 | 展示人物方向壁纸 | `categories=001` |
| `/collection` | 我的收藏 | 展示本地收藏记录 | 本地存储 |
| `/download` | 下载中心 | 展示下载任务、完成记录和下载目录 | 本地存储 + Tauri |
| `/downloads` | 重定向 | 兼容旧入口 | 重定向到 `/download` |
| `/about` | 关于 | 应用信息页面 | 静态内容 |

左侧菜单分组：

- 在线壁纸：热门推荐、动漫精选、人物精选
- 我的壁纸：我的收藏、下载中心
- 其他：关于

## 3. 浏览与筛选

热门、动漫、人物页面共用 `useBrowsePage` 组合式逻辑。每个页面提供不同的默认查询参数，切换菜单时会重置壁纸 store 并重新加载对应类型图片。

浏览流程：

1. 页面挂载时重置 `wallpaper` store。
2. 使用页面默认参数请求 `/search`。
3. 用户调整顶部筛选项或搜索关键词。
4. 页面合并筛选参数并重新请求第一页。
5. 滚动到底部时触发下一页加载，并追加到当前列表。

当前筛选能力：

| 筛选项 | 说明 |
| --- | --- |
| 搜索关键词 | 写入 `q` 参数，支持 Wallhaven 搜索语法 |
| 排序 | 支持热门、最新、浏览量、收藏数、随机等排序 |
| 纯净度 | 支持 SFW、Sketchy、NSFW 参数组合 |
| 最低分辨率 | 支持全部、1080p、2K、4K、8K 等选项 |

数据入口：

- 前端 API 封装：`src/api/wallhaven.ts`
- Tauri 代理命令：`fetch_wallhaven_api`
- 状态存储：`src/stores/wallpaper.ts`

## 4. 壁纸卡片

壁纸网格以卡片形式展示缩略图。卡片承担浏览页的主要识别和操作入口。

卡片能力：

- 展示缩略图、分辨率、文件大小等核心信息。
- 鼠标悬浮时出现轻量毛玻璃操作层。
- 支持收藏/取消收藏。
- 支持下载。
- 支持点击进入大图详情。

交互反馈：

- 收藏成功、取消收藏会通过全局 Toast 给出反馈。
- 下载操作会进入下载中心状态流。
- 悬浮层的模糊强度经过降低，保证用户仍能隐约看到小图内容。

## 5. 大图详情

点击图片后进入详情弹窗。详情弹窗重点优化了大图加载体验和竖版图片查看体验。

当前能力：

- 大图请求完成前，先用缩略图作为过渡背景。
- 大图加载成功后替换为高清图。
- 竖版图片使用完整包含布局，避免被裁切。
- 支持放大/还原。
- 放大后支持鼠标拖动查看图片不同区域。
- 操作栏提供收藏、下载、放大、关闭等动作。
- 右侧详情信息使用透明毛玻璃样式，能隐约看到下方图片列表。

实现要点：

- 高清图片通过 `fetch_wallpaper_image` 由 Rust 后端下载到本地缓存，再由 `convertFileSrc` 转换给前端显示。
- 缩放状态由 `zoomed` 控制。
- 拖动偏移由 `panX`、`panY` 控制。
- 弹窗关闭或切换图片时会重置加载、缩放和拖动状态。

## 6. 收藏

收藏功能以本地存储为主，不依赖远端账号。

当前能力：

- 浏览卡片和详情弹窗中均可收藏/取消收藏。
- 收藏页面展示已收藏图片。
- 收藏状态会在卡片与详情弹窗中同步体现。
- 操作完成后使用同一套 Toast UI 给出结果反馈。

存储位置：

- Pinia store：`src/stores/collection.ts`
- LocalStorage key：由收藏 store 管理

## 7. 下载中心

下载中心负责下载任务、下载完成记录、本地文件操作和下载目录设置。

当前能力：

- 设置下载目录。
- 打开下载目录。
- 发起壁纸下载。
- 展示下载中任务。
- 取消下载中任务，并立即从列表移除。
- 展示已完成下载记录。
- 已完成记录左侧显示缩略图，便于识别图片内容。
- 已完成记录右侧提供文件夹按钮，可定位到本地图片文件。
- 删除已完成记录时，在删除按钮附近显示确认浮层。
- 确认删除后删除本地文件，并从列表移除。

下载状态来源：

- 前端发起：`download_image`
- 后端进度事件：`download-progress`
- 后端错误事件：`download-error`
- 本地记录：`wallhaven-downloads`
- 下载目录：`wallhaven-download-dir`

关键 Tauri 命令：

| 命令 | 用途 |
| --- | --- |
| `download_image` | 后台下载图片 |
| `cancel_download` | 标记取消并移除活跃任务 |
| `delete_download_file` | 删除本地图片文件并清理完成记录 |
| `get_download_dir` | 获取当前下载目录 |
| `set_download_dir` | 设置并创建下载目录 |
| `choose_download_dir` | 打开系统目录选择器 |
| `open_download_folder` | 打开下载目录 |
| `reveal_download_file` | 在系统文件管理器中定位图片 |

## 8. Toast 反馈

全局 Toast 用于轻量反馈，不打断用户浏览。

当前使用场景：

- 收藏成功。
- 取消收藏。
- 后续可扩展到下载失败、删除失败、目录选择失败等结果提示。

设计特点：

- 使用 `toast-glass` 样式，保持和整体毛玻璃 UI 一致。
- 独立 `ToastViewport` 组件挂载在应用框架中。
- 状态由 `src/stores/toast.ts` 管理。

## 9. Tauri 后端职责

Rust 后端主要负责桌面能力与网络代理。

后端职责：

- 代理 Wallhaven API 请求，避免前端 CORS 问题。
- 代理高清图片下载，避免 CDN 防盗链导致 403。
- 管理图片缓存目录。
- 管理下载任务状态。
- 发送下载进度事件。
- 打开系统目录选择器。
- 打开下载目录和定位本地文件。
- 删除本地下载文件。

前端应避免直接承担系统文件能力，相关能力优先通过 Tauri command 暴露。

## 10. 数据持久化

当前主要使用 LocalStorage 保存轻量状态：

| 数据 | 说明 |
| --- | --- |
| 收藏记录 | 保存用户收藏的壁纸 |
| 下载记录 | 保存最近下载任务和完成记录，最多保留一部分历史 |
| 下载目录 | 保存用户选择的下载位置 |

高清图片缓存和下载文件由 Tauri/Rust 落到本地文件系统。

## 11. 维护建议

- 新增浏览分类时，优先复用 `useBrowsePage`，只调整默认查询参数和路由菜单。
- 新增用户反馈时，优先复用 Toast store，保持反馈方式统一。
- 新增下载相关能力时，前端只维护 UI 状态，文件系统操作交给 Tauri command。
- 涉及 Wallhaven 参数时，优先参考 `docs/wallhaven-api.md`。
- 涉及视觉调整时，优先参考 `docs/ui-design-system.md` 和 `src/style.css`。
